</br>
動画一括変換ツール（30fps＋GIF＋PNG連番）取扱説明</br>
1. このアプリでできること</br>
</br>
このアプリは、指定したフォルダの中にある動画ファイルをまとめて処理します。対象になるのは次の拡張子です。</br>
</br>
.avi</br>
.mov</br>
.mp4</br>
</br>
処理内容は3種類あります。ボタン横のチェックでON/OFFできます。</br>
MP4を30fpsで書き出す</br>
元の動画を30fpsにそろえたMP4として出力します。</br>
「同じフレーム数」か「同じ時間長」かを選べます。</br>
出力先は元動画と同じ場所にできる _30fpsMP4 フォルダ です。</br>
GIFアニメを作る</br>
元の動画からGIFを作ります。</br>
GIFの大きさは「GIF倍率」で調整します（例：0.2 → 20%縮小）。</br>
出力先は _GifAnim フォルダ です。</br>
連番PNGを書き出す（デフォルトでON）</br>
元の動画を30fpsでPNG連番にします。</br>
出力先は _PngSeq フォルダ の中に、動画ごとのサブフォルダを作って保存します。</br>
例）C:\movie\cat.mp4 →</br>
C:\movie\_PngSeq\cat\cat_0001.png</br>
C:\movie\_PngSeq\cat\cat_0002.png …</br>
何回も同じ動画を変換すると同じ番号は上書きされます（続き番号にはなっていません）。</br>
さらに、GIFを書き出したときだけ、同じフォルダに _index.html を作って、GIFの一覧を見られるようにできます（「HTMLを生成する」がONのとき）。</br>
</br>
2. 動作に必要なもの</br>
Windows で動作する .NET / WinForms アプリ</br>
同じ場所に ffmpeg.exe があること</br>
→ このアプリは内部で ffmpeg を呼び出しているだけなので、ffmpeg が無いと変換は始まりません</br>
→ いまの設定ではコーデックを細かく指定していないので、普通の Windows 用 ffmpeg ならそのまま動きます</br>
</br>
3. 画面の見方</br>
(1) フォルダ選択</br>
「フォルダ選択」ボタンを押すと処理する親フォルダを指定できます。</br>
下のテキストボックスにパスが入ります。</br>
このパスを起点にファイルを探します。</br>
(2) 「サブフォルダを含む」</br>
ONにすると、選んだフォルダの中の全部の下階層をたどって動画を探します。</br>
OFFにすると、その階層だけが対象です。</br>
(3) 進行状況バー</br>
見つかった動画の本数を上限にして、1本終わるごとに増えていきます。</br>
中止した場合は途中の位置で止まります。</br>
(4) 30fpsのそろえ方</br>
「同じフレーム数」 … フレーム数を優先して30fpsに合わせます。短くなったり長くなったりしないように、setpts をかけています。</br>
「同じ時間長」 … 再生時間を優先して30fpsにします。もとの長さを保ちたいときはこちら。</br>
この2つの切り替えはMP4出力にだけ影響します。PNGやGIFはそれぞれの処理でfps=30をかけています。</br>
(5) 出力項目</br>
MP4を出力</br>
→ ONのときだけ _30fpsMP4 に書き出し</br>
→ 出力ファイル名は「元ファイル名 + _30fps.mp4」</br>
</br>
GIFを出力</br>
→ ONのときだけ _GifAnim に書き出し</br>
→ 出力ファイル名は「元ファイル名 + _30fps.gif」</br>
HTMLを生成する</br>
→ GIF出力がONのときだけ有効になります</br>
→ 変換終了後に、指定したフォルダの直下に _index.html を作って、生成したGIFを並べたページを作ります</br>
連番PNG書き出し</br>
→ デフォルトでON</br>
→ _PngSeq/<元ファイル名>/ の中に <元ファイル名>_%04d.png で書き出します</br>
→ 30fps固定です</br>
(6) GIF倍率</br>
数字を小さくするとGIFのサイズを小さくできます。</br>
0.2 なら 20%、0.5 なら 50%、1.0 なら等倍です。</br>
0 やマイナスを入れるとエラーにして止めるようにしてあります。</br>
(7) 実行 / 中止</br>
**「実行」**を押すと、設定に従って順番に変換を始めます。</br>
**「中止」**は変換中だけ押せます（有効になります）。</br>
押した時点で内部で動いている ffmpeg プロセスを全部止めます。</br>
その時点までに書き出されたファイルは残ります。</br>
途中で止めたので、PNGの連番が途中で終わっていることがあります。</br>
</br>
4. 出力されるフォルダとファイル名</br>
MP4</br>
出力場所: 元フォルダ/_30fpsMP4/</br>
ファイル名: 元ファイル名_30fps.mp4</br>
GIF</br>
出力場所: 元フォルダ/_GifAnim/</br>
ファイル名: 元ファイル名_30fps.gif</br>
PNG連番</br>
出力場所: 元フォルダ/_PngSeq/元ファイル名/</br>
ファイル名: 元ファイル名_0001.png, 元ファイル名_0002.png, …</br>
4桁ゼロ埋め（%04d）</br>
HTML</br>
出力場所: 元フォルダ/_index.html</br>
内容: そのときに変換したGIFの一覧を並べた簡単なページ</br>
</br>
5. 処理の流れ（実行中に何が起きるか）</br>
「実行」ボタンを押す</br>
指定フォルダをスキャンして対象の動画リストを作る</br>
動画を1本ずつ処理していく</br>
MP4がONならまずMP4を作る</br>
GIFがONならGIFを作る</br>
PNGがONならPNG連番を吐く</br>
1本終わるたびに進捗バーが進む</br>
全部終わったら「変換が終了しました。」を表示</br>
GIF＋HTMLの場合は最後に _index.html を書く</br>
※「中止」を押すと3番の途中で止まります。</br>
</br>
6. こういうときは？</br>
Q1. 「MP4が出ない」って出るのは？</br>
今回の状態では、余計なコーデック指定を外してあります。なので</br>
ffmpeg.exe が無い</br>
ffmpeg.exe にパスが通っていない</br>
出力フォルダに書き込めない</br>
のどれかが原因になることが多いです。</br>
まずは実行フォルダに置いた ffmpeg.exe が起動できるか（コマンドプロンプトから ffmpeg -version で出るか）を見てください。</br>
Q2. PNGを別ドライブに書き出したい</br>
今は「元動画と同じ場所に _PngSeq を作る」固定です。可変にする場合はUIを1個増やす必要があります。</br>
Q3. PNGを続き番号にしたい</br>
今は毎回 xxx_0001.png から書き直します。続き番号にするには、処理の前にそのフォルダを見に行って最大番号を取るロジックを入れる必要があります。</br>
今の説明書はそこまでの実装を前提にしていません。</br>
Q4. GIFを作るときにもっときれいにしたい</br>
今は簡単な scale=... だけです。パレット作成＋2passにすればきれいになりますが、その場合は処理を増やします。</br>
</br>
7. 注意点</br>
変換中にアプリを閉じると、そのとき動いているffmpegも止めるようにしていますが、まれにプロセスが残ることがあります。</br>
気になったらタスクマネージャーで ffmpeg.exe が残っていないか確認してください。</br>
同じファイルを何度も変換すると、同じ名前の出力は上書きされます。バックアップが必要な場合は先にコピーしてください。</br>
大きい動画をたくさん置いたフォルダに対して「サブフォルダを含む」をONでやると時間がかかります。途中でやめたいときは中止ボタンで止めてください。</br>
